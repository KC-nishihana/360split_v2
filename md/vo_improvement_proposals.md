## A. 精度向上系

### A1. ORB/AKAZE特徴量のハイブリッド検出

**現状の課題**: `goodFeaturesToTrack` (Shi-Tomasi Corner) は低テクスチャ領域やモーションブラーに弱い。
**提案**: ORBまたはAKAZEとのハイブリッド検出を導入し、シーン特性に応じて切り替える。

- 低テクスチャ検出時 → AKAZEにフォールバック（ブロブ特徴量で補完）
- 回転が大きい区間 → ORB（回転不変性あり）に切り替え
- 設定: `VO_FEATURE_METHOD: "auto" | "klt" | "orb" | "akaze"`

**期待効果**: 特徴点追跡失敗率の低減（特に屋外撮影の空・地面領域）

### A2. 5点法 → MAGSAC++ へのアップグレード

**現状の課題**: `findEssentialMat` の RANSAC は外れ値が多い場合に不安定。
**提案**: OpenCV 4.x の `USAC_MAGSAC` をオプション提供。

- `VO_ESSENTIAL_METHOD: "ransac" | "magsac"`
- MAGSACはマージナルな外れ値（準外れ値）のハンドリングが優れている
- 自動切り替え: インライア比率 < 30% → MAGSACにスイッチ

**期待効果**: `vo_inlier_ratio` の底上げ、`estimate_failed_or_low_inlier` 発生率の削減

### A3. サブピクセル精度光学フロー

**現状の課題**: KLT追跡はピクセル精度に依存。
**提案**: 追跡後に `cornerSubPix` でリファインし、サブピクセル精度を確保。

- 計算コスト: フレームあたり +2〜5ms 程度
- 設定: `VO_SUBPIXEL_REFINE: bool = True`

**期待効果**: 回転推定精度の向上（特に微小運動時）

## C. パフォーマンス・効率系

### C1. GPUアクセラレーション（CUDA光学フロー）

**現状の課題**: 大量フレーム処理時のVO計算がボトルネック。
**提案**: NVIDIA GPU環境で `cv2.cuda` の光学フロー計算を利用。

- `cv2.cuda.SparsePyrLKOpticalFlow` でKLTをGPU実行
- CPU/GPU自動選択: CUDA利用可能なら自動でGPU使用
- 設定: `VO_USE_GPU: "auto" | "cpu" | "cuda"`

**期待効果**: CUDA環境で2〜5倍の速度向上（Stage0の全フレームスキャンで顕著）

### C2. Apple Silicon MPS対応検討

**現状の課題**: macOS MPS環境では現状CPUフォールバックのみ。
**提案**: CoreMLまたはMetal Performance Shadersでの光学フロー計算。

- VisionFramework の `VNGenerateOpticalFlowRequest` を利用
- pyobjcブリッジ経由、またはサブプロセスでSwift CLIを呼び出し

**期待効果**: Apple Silicon Macユーザーの処理速度改善

### C3. キーフレーム間適応的サブサンプリング

**現状の課題**: `VO_FRAME_SUBSAMPLE` は固定間隔で、高速移動区間で精度が落ちる。
**提案**: 直前フレームの光学フロー量に基づいてサブサンプリング間隔を動的調整。

- 高フロー（大きな動き）→ サブサンプル間隔を狭める（毎フレームVO）
- 低フロー（静止に近い）→ 間隔を広げる（N フレームスキップ）
- 設定: `VO_ADAPTIVE_SUBSAMPLE: bool = True`

**期待効果**: 計算量を抑えつつ、重要な区間での軌跡精度を維持

## D. 堅牢性・信頼性向上系

### D1. VO信頼度スコアの導入

**現状の課題**: `vo_valid` は二値判定のみで、推定品質のグレーゾーンを表現できない。
**提案**: 複合的な信頼度スコア `vo_confidence ∈ [0, 1]` を導入。

- 構成要素:
  - インライア比率（0〜1） × 重み0.4
  - 特徴点分布の均一性（画像4象限の分散） × 重み0.2
  - ポーズの物理的妥当性（回転<30°/フレーム、並進の連続性） × 重み0.2
  - 追跡特徴点数の充足率 × 重み0.2
- Stage3の軌跡一貫性スコアに信頼度で重み付け

**期待効果**: 「推定は成功したが信頼できない」ケースの適切な扱い

### D2. ループ検出・ドリフト補正

**現状の課題**: 長い動画でVO軌跡がドリフトし、後半フレームのスコアが不安定になる。
**提案**: 簡易的なBag-of-Wordsベースのループ検出とポーズグラフ最適化。

- 一定間隔でORB特徴量のBoW記述子を蓄積
- 類似フレーム検出時にループ制約を追加
- g2o / Ceres Solver によるポーズグラフ最適化（オプション依存ライブラリ）
- 設定: `VO_LOOP_CLOSURE: bool = False`（デフォルトOFF、上級者向け）

**期待効果**: 5分以上の長尺動画での軌跡品質改善

### D3. マルチスケール特徴追跡

**現状の課題**: `VO_DOWNSCALE_LONG_EDGE` で一律縮小すると近景・遠景の特徴バランスが崩れる。
**提案**: 2段階の解像度で特徴点を検出し統合。

- 全体（縮小版）: 大局的な動き推定
- ROIクロップ（原寸）: 精密な回転推定
- 両方の特徴点を統合してEssential Matrixを推定

**期待効果**: 速度を維持しつつ、特に回転推定の精度を改善

### E1. Rerun連携強化（閲覧・分析特化）

既存の `RerunKeyframeLogger` を拡張し、VO品質の分析ビューを追加する。

#### E1-a. VO信頼度ヒートマップ軌跡

- 軌跡の各ポイントを `vo_confidence` で色分け（緑→黄→赤）
- VO失敗区間を破線 or 灰色で表示
- Rerun `LineStrips3D` の color属性を活用

#### E1-b. メトリクスタイムラインの拡充

- 現状: flow magnitude, match count のみ
- 追加: `vo_confidence`, `inlier_ratio`, `rotation_delta_deg`, `step_proxy_norm`
- Rerun `TimeSeriesScalar` で各メトリクスを個別プロットし、タイムライン連動

#### E1-c. Blueprint プリセット

- 分析目的別のRerun Blueprint（レイアウト定義）を同梱
  - `vo_debug.blueprint`: 3D軌跡 + メトリクス4段 + フレーム画像
  - `keyframe_review.blueprint`: キーフレーム画像 + スコア + 軌跡上マーカー
  - `trajectory_overview.blueprint`: フル軌跡 + 速度グラフ

#### E1-d. Stage0 VO進捗のリアルタイムストリーミング

- 現状: Stage2/3のみリアルタイムログ
- 拡張: Stage0のVO推定もフレームごとにストリーミング
- 処理中にキャリブレーションエラーや特徴点不足を即座に検出可能

**実装コスト**: 低〜中（既存logger拡張）
**期待効果**: VO品質ダッシュボードの大部分をカバー

### E2. GUI操作性強化（操作・編集特化）

既存のPySide6 GUIに、VO情報を活用したインタラクティブ機能を追加する。

#### E2-a. キーフレーム手動追加・除外

- TimelineWidget上でフレームを右クリック → 「キーフレームに追加/除外」
- KeyframeListWidget で複数選択 → 一括除外
- 変更後、Stage3スコアのリアルタイム再計算（Live Preview拡張）

#### E2-b. VO信頼度オーバーレイ

- TimelineWidgetに `vo_confidence` ラインを追加（半透明背景帯）
- 低信頼度区間をオレンジ/赤の帯でハイライト
- 「この区間はVO推定が不安定です」ツールチップ

#### E2-c. 軌跡ベースのギャップ検出・提案

- VO軌跡上で、キーフレーム間距離が大きすぎる区間を自動検出
- 「フレーム 234-456 間にキーフレームが不足しています。追加を推奨」通知
- ワンクリックで推奨フレームを追加

#### E2-d. VO設定プリセット

| プリセット名 | 用途 | 主な設定 |
|-------------|------|---------|
| Quick | プレビュー・素早い確認 | サブサンプル=3, 特徴点=300, ダウンスケール=640 |
| Balanced | 通常利用 | 現在のデフォルト値 |
| Precise | 高精度が必要な場面 | サブサンプル=1, 特徴点=1000, MAGSAC, サブピクセル有効 |

- SettingsPanelにプリセットドロップダウンを追加
- プリセット選択時、対応パラメータを一括変更
- カスタム設定の保存・読み込み対応

**テーマ**: 最小コストでVO品質とUXの土台を作る

```
D1. VO信頼度スコア     ← 全ての改善の基盤。最優先
 ↓
A2. MAGSAC++           ← API切替のみ。D1と同時期に実装可
 ↓
A3. サブピクセル精度    ← 1行追加レベル。A2と一緒に入れる
 ↓
C3. 適応的サブサンプリング ← Stage0の効率改善
 ↓
E2-d. VOプリセット     ← Phase1の成果をユーザーに届ける
```

**成果物**: vo_confidence がパイプライン全体で使えるようになり、MAGSAC/サブピクセルの「Precise」プリセットが選択可能に。

### Phase 2: 可視化・操作性（2〜4週間）

**テーマ**: D1の信頼度スコアを活用した可視化と、キーフレーム編集のインタラクション

```
E2-b. VO信頼度オーバーレイ（GUI Timeline）
E1-a. 信頼度ヒートマップ軌跡（Rerun 3D）
E1-b. メトリクスタイムライン拡充（Rerun）
 ↓
E2-a. キーフレーム手動追加・除外（GUI）  ← ユーザー操作の幅を広げる
E2-c. ギャップ検出・提案（GUI）
```

**成果物**: GUIで「どの区間のVOが不安定か」が一目でわかり、キーフレームの手動調整が可能に。Rerunでは3D軌跡上で品質が色で可視化。

### Phase 3: 精度・性能の深化（1〜2ヶ月）

**テーマ**: 対応シーンの拡大とパフォーマンス改善

```
A1. ハイブリッド特徴検出   ← 低テクスチャ対応
D3. マルチスケール追跡     ← 近景・遠景バランス
E1-c. Blueprint プリセット ← Phase2の可視化を整理
```
